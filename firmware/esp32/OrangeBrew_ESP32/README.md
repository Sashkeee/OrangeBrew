# Прошивка OrangeBrew ESP32 (HIL-Стенд)

Этот скетч превращает плату ESP-32 в аппаратный модуль управления и считывания датчиков для проекта OrangeBrew.

## 📦 Необходимые библиотеки
Для компиляции в Arduino IDE перейдите в `Скетч -> Подключить библиотеку -> Управлять библиотеками...` (или `Ctrl+Shift+I`) и установите:
1. **OneWire** (автор: Paul Stoffregen)
2. **DallasTemperature** (автор: Miles Burton)
3. **ArduinoJson** (версия 6.x.x или 7.x.x, автор: Benoit Blanchon)

## 📋 Описание работы
1. **Связь**: ESP32 общается с OrangeBrew (Node.js) по Serial-порту (USB) на скорости `115200` бод.
2. **ТЭН (Лампочка 1)**: Используется *"Медленный ШИМ"* (Software PWM) с окном в 2 секунды. Это сделано специально для твердотельных реле (SSR), обладающих встроенным Zero-Cross (переключение при переходе через ноль). Обычный быстрый `analogWrite` с ними не работает.
3. **Насос (Лампочка 2)**: Обычный дискретный пин (ВКЛ / ВЫКЛ).
4. **Датчики DS18B20**: Каждую секунду микроконтроллер асинхронно опрашивает все термометры на 1-Wire шине и отправляет в монитор порта строку вида:
`{"type":"sensors_raw","sensors":[{"address":"2861e61s0a0000bc","temp":24.5}, ...]}`

## 🔌 Подключение (Пины)
По умолчанию в коде заданы:
*   Датчики DS18B20 (линия DATA): `GPIO 4`
*   Реле ТЭНа: `GPIO 25`
*   Реле Насоса: `GPIO 26`
Пины легко меняются в заголовке `OrangeBrew_ESP32.ino`.

### Утилита идентификации датчиков
Так как датчиков несколько и все они висят параллельно на одном пине (OneWire), бэкенд проекта OrangeBrew получит их уникальные 8-байтные HEX адреса. Мы сделаем в веб-интерфейсе "Мастер настройки датчиков", где вы сможете приложить руку к конкретному датчику, увидеть, как на графике выросла температура именно у адреса `28XX...` и присвоить ему роль "Куб" или "Колонна".
