# Топ-10 рекомендованных навыков (SkillsMP) для проекта OrangeBrew

Этот список был составлен на основе анализа архитектуры проекта OrangeBrew (ESP32 на C++, Node.js бэкенд с SQLite и SerialPort, React-фронтенд с WebSockets и графиками Recharts, Telegram-уведомления). Данные навыки (формат `SKILL.md`) помогут ИИ-ассистентам писать более надежный, оптимизированный и безошибочный код для подобных систем.

## ⚙️ Железо и Микроконтроллеры (ESP32)

1. **`cpp-arduino-nonblocking`** (Неблокирующий код C++)
   Навык, исключающий использование функции `delay()`. Заставляет строить архитектуру прошивки на базе таймеров `millis()`, прерываний (ISR) и конечных автоматов (State Machines). Критически важно для ПИД-регулятора, чтобы ESP32 не зависал от чтения Serial.

2. **`esp32-freertos-multitasking`**
   Скилл для продвинутой работы с ядрами ESP32. Позволяет грамотно разделить задачи: например, вычисления ПИД для ТЭНа на первом ядре, а обработку Serial и опрос датчиков Dallas DS18B20 — на втором.

## 🔌 Бэкенд и Интеграции (Node.js)

3. **`node-serialport-robust`** (Отказоустойчивость COM-портов)
   Специфичный навык для работы с железом. Автоматизирует добавление обработчиков событий `error`, `disconnect` и логики авто-переподключения, предотвращая падение сервера (Node.js) при физическом отключении USB-кабеля или рестарте ESP32.

4. **`telegram-bot-polling-master`**
   Навык проектирования Telegram-ботов без "зависаний". Учит ИИ правильно отправлять фоновые отчеты (например, о температурах варки) без блокировки основного потока `Express` и Event Loop'а.

5. **`sqlite-architecture-migrations`**
   Скилл, заставляющий ИИ всегда писать безопасные миграции баз данных через `PRAGMA foreign_keys=off` при изменении таблиц (особенно `CHECK constraint`), вместо того чтобы пытаться выполнить неподдерживаемые варианты `ALTER TABLE` в SQLite.

## 🖥️ Фронтенд (React)

6. **`react-recharts-expert`** (Продвинутые графики)
   Помогает проектировать сложные дашборды: автоматическое зумирование графиков процессов (дистилляция, затирание), оптимизация рендеринга больших массивов данных (до 10 000 точек) без тормозов и багов с осью времени.

7. **`react-websockets-sync`** (Live-синхронизация)
   Навык для работы с `liveServer.js` и State-менеджментом. Внедряет строгую логику переподключения (`reconnect`) WebSocket-соединений на клиенте и обработки ситуаций потери связи с сервером оборудования.

## 📐 Математика и Физика процесса

8. **`pid-tuning-algorithms`** (Инжиниринг ПИД-регуляторов)
   Встроенные знания классических методов настройки (Зиглер-Никольс, релейный метод автотюнинга). Помогает ИИ писать сложную физико-математическую логику без багов при разработке или тюнинге "калибратора ПИД" куба.

9. **`industrial-state-machines`** (Промышленные стейт-машины)
   Навык для жесткого и безопасного описания последовательности технологических процессов (Затирание ➡️ Паузы ➡️ Кипячение ➡️ Охлаждение). Использование паттернов состояния (например, `XState`), защищающих от несанкционированных переходов между стадиями.

## 🧪 Тестирование

10. **`playwright-ui-testing`** (Автотесты интерфейсов)
    Шаблоны Playwright для написания E2E (End-to-End) тестов. Позволяет ИИ автоматически генерировать автотесты, где робот-браузер сам проходит полный путь пользователя (выбор рецепта, запуск варки, проверка отрисовки графиков и корректного завершения).
