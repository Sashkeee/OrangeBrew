Искусство Тестирования

Полное руководство по автоматизации контроля качества в проекте OrangeBrew. От простых функций до сложных интерфейсов.

Архитектура Качества

🧪

Unit Tests (Юниты)

Проверка элементарной логики. Математика, обработка строк, форматирование дат. Быстрые, надежные, не требуют браузера.

🧱

Component Tests

Проверяем кирпичики интерфейса. Отрисовывается ли кнопка? Меняется ли цвет при наведении? Появляется ли текст ошибки?

🔄

Integration Tests

Проверка взаимодействия страницы с данными. Как страница варки реагирует на сигнал от датчика через хук? Идет ли запрос на сервер?

Как читать тестовый код

example.test.js 
// 1. Группировка тестов по теме
describe('Функция форматирования', () => {

  // 2. Описание конкретного сценария
  it('должна превращать 90 секунд в 01:30', () => {
    
    // 3. Само сравнение (Assertion)
    // Ожидаю, что результат format(90) СООТВЕТСТВУЕТ "01:30"
    expect(formatTime(90)).toBe('01:30');
  });

});

Основные инструменты:

1
render() — виртуально отрисовывает ваш React-компонент, чтобы тест мог его "увидеть".
2
screen — ваши глаза. Позволяет искать элементы: screen.getByText('ОК').
3
fireEvent — ваши руки. Нажимает на кнопки, вводит текст: fireEvent.click(button).

Специфика OrangeBrew

Mocks (Заглушки) — наш главный секрет

Наши тесты не подключаются к настоящему ESP32. Мы используем vi.mock(), чтобы "обмануть" компоненты. Мы говорим им: "Считай, что сейчас температура 65°C и сервер доступен". Это позволяет тестировать UI в идеальных, предсказуемых условиях.

// Имитируем работу датчиков
vi.mock('../hooks/useSensors', () => ({
    useSensors: () => ({ 
        sensors: { boiler: { value: 100 } } 
    })
}));

3 шага к запуску

Консоль

Просто напиши npm test в корне фронтенда.

UI-режим

Напиши npx vitest --ui для красивого интерфейса в браузере.

Автоматика

GitHub Actions сам запустит тесты при каждом твоем коммите.

Создано специально для команды OrangeBrew. Качество — это не случайность, а результат тестирования.
